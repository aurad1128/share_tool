# 开发日志目录

本目录用于记录每天的开发进度、完成事项和待办事项。

## 日志命名规范

文件命名格式：`YYYY-MM-DD.md`

例如：`2026-05-16.md`

## 日志模板

每天的日志应包含以下内容：

1. **日期和星期**
2. **当前阶段**
3. **今日目标**
4. **完成事项**
5. **遇到的问题**
6. **解决方案**
7. **明日计划**
8. **工作时长**

## 使用说明

- 每天开始工作前创建当天的日志文件
- 工作过程中随时更新完成事项
- 遇到问题及时记录
- 每天结束时总结并规划明天的任务
