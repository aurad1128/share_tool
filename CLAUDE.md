```

```

## ⚠️ 重要原则

### 开发原则
1. **不要一口气做太多** - 每个阶段完成后测试
2. **稳定安全推进** - 确保当前功能稳定再进入下一阶段
3. **及时记录** - 遇到问题立即记录到开发日志
4. **代码质量** - 保持代码简洁可维护

### 测试原则
1. 每个功能开发完立即测试
2. 每个阶段完成后进行集成测试
3. 发现问题立即修复，不要积累

### 文档原则
1. 每天更新开发日志
2. 重要决策记录到日志
3. 问题和解决方案详细记录
